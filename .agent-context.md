# The RUT Universe Context

## Philosophy: The Path is the Point
> "You don’t pick the path because of the destination; you discover the destination because of the path."

When building or discussing a **Really Useful Thing (RUT)** within the CharityOps ecosystem, adhere to the following core tenets:

1. **We don't build for the Destination**
   Traditional software is built for a finished, static product. A RUT is the path. It is built in 24 hours to solve what is hurting *right now*. It doesn't have every feature because you don't need every feature to take the next step.
   
2. **"Useful" is what keeps you moving**
   A tool isn't useful because it is "very good" or "complex." It is useful because it removes an obstacle in your way.
   - A *90-day lifespan* isn't a limitation; it’s a commitment to stay lean.
   - *WhatsApp integration* isn't a compromise; it’s meeting users where their feet are already planted.
   - *Single-function design* isn't a lack of ideas; it’s a focus on the task at hand.

3. **Discovery through Experimentation**
   Follow the **"Fail on Purpose"** mindset. A "Useful" tool that helps today is better than a "Perfect" tool that never arrives. We invite exploration rather than delivering a finished product.
